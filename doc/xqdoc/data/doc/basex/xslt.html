Functions

processor()
Returns:
xs:string
Description: Returns the name of the applied XSLT processor, or the path to a custom implementation (currently: "Java", "Saxon EE", "Saxon PE", or "Saxon HE").

transform($input, $stylesheet)
Arguments:
$input item()
$stylesheet item()
Returns:
node()
Description: Transforms the document specified by $input , using the XSLT template specified by $stylesheet , and returns the result as node. $input and $stylesheet can be specified as
  • xs:string, containing the path to the document,
  • xs:string, containing the document in its string representation, or
  • node(), containing the document itself.

The $params argument can be used to bind variables to a stylesheet, which can either be specified

  • as children of an <xslt:parameters/> element; e.g.: 
 <xslt:parameters> <xslt:key1 value='value1'/> ... </xslt:parameters>
  • or as map, which contains all key/value pairs: 
 map { "key1": "value1", ... }

Note that only strings are supported when using Saxon (XSLT 2.0).
